# MODERN LABEL - E-Commerce Streetwear Web Application

โปรเจกต์นี้เป็นเว็บไซต์ร้านค้าออนไลน์ (E-Commerce) สำหรับแบรนด์แฟชั่นสตรีทแวร์แนวพรีเมียม ชื่อว่า **MODERN LABEL** พัฒนาขึ้นโดยใช้ Vue 3 และ Vite พร้อมกับการดีไซน์สไตล์มินิมอลและรองรับการแสดงผลแบบ Responsive เต็มรูปแบบ

---

## 🛠️ เทคโนโลยีและเครื่องมือที่ติดตั้งในโปรเจกต์ (Dependencies)

โปรเจกต์นี้ขับเคลื่อนด้วยไลบรารีและเทคโนโลยีหลักดังต่อไปนี้:

### 1. **Core Framework & Build Tools**
*   **Vue 3 (`vue`: `^3.5.34`)**: เฟรมเวิร์กหลักที่ใช้พัฒนาเว็บแบบ Single Page Application (SPA) ด้วยฟีเจอร์ Composition API (`<script setup>`)
*   **Vite (`vite`: `^8.0.12`)**: เครื่องมือสำหรับ Bundle และรันเซิร์ฟเวอร์พัฒนาที่มีความเร็วสูงและสนับสนุน Hot Module Replacement (HMR)
*   **@vitejs/plugin-vue (`^6.0.6`)**: ปลั๊กอินอย่างเป็นทางการของ Vite เพื่อสนับสนุนการคอมไพล์ไฟล์ Vue Single File Components (SFCs)

### 2. **Styling & Components**
*   **Bootstrap 5 (`bootstrap`: `^5.3.8`)**: CSS Framework สำหรับจัดการโครงสร้าง Grid, เลย์เอาต์ และดีไซน์เริ่มต้นที่รองรับ Responsive
*   **Bootstrap Vue Next (`bootstrap-vue-next`: `^0.45.6`)**: ชุดคอมโพเนนต์สำหรับ Vue 3 ที่แปลงคอมโพเนนต์เดิมของ Bootstrap 5 ให้ใช้งานง่ายขึ้นในรูปแบบ Vue Components (เช่น `BOffcanvas`)
*   **Vanilla CSS (`src/style.css`)**: มีการสร้างระบบ Design Token, ธีมสี (Light & Dark), อนิเมชันการเลื่อน, เงา และปุ่มกดแนวพรีเมียมเฉพาะตัวแบบ Custom

### 3. **Routing**
*   **Vue Router 4 (`vue-router`: `^4.6.4`)**: ระบบเราเตอร์เพื่อนำทางไปยังหน้าต่าง ๆ ของเว็บไซต์อย่างราบรื่น (เช่น หน้าหลัก หน้าหมวดหมู่สินค้า หน้าดูรายละเอียดสินค้า)

## 🚀 วิธีการรันโปรเจกต์

หากต้องการเริ่มต้นใช้งานและพัฒนาโปรเจกต์ต่อ ให้ปฏิบัติตามคำสั่งต่อไปนี้:

### 1. ติดตั้ง Dependencies ทั้งหมดในครั้งแรก
```bash
npm install
```

### 2. รันแอปพลิเคชันสำหรับโหมดพัฒนา (Development Mode)
```bash
npm run dev
```
ระบบจะเปิดเซิร์ฟเวอร์พัฒนาที่ปกติจะเป็น [http://localhost:5173](http://localhost:5173)

### 3. ตรวจสอบความถูกต้องและสร้างเวอร์ชันสำหรับการใช้งานจริง (Production Build)
```bash
npm run build
```
ไฟล์สำหรับขึ้นระบบจริงจะถูกสร้างขึ้นในโฟลเดอร์ `/dist`

## 🐙 วิธีใช้งาน Git เบื้องต้น (Clone, Pull, Merge)

สำหรับสมาชิกทีมที่ต้องการร่วมพัฒนาโปรเจกต์ สามารถใช้คำสั่ง Git พื้นฐานเพื่อทำงานร่วมกันได้ดังนี้:

### 1. การดาวน์โหลดโปรเจกต์ลงเครื่องครั้งแรก (Clone)
ใช้ดาวน์โหลดโฟลเดอร์โค้ดจากระบบเซิร์ฟเวอร์กลาง (Remote Repository) มาติดตั้งที่เครื่องคอมพิวเตอร์ของคุณ:
```bash
git clone <URL_ของ_Repository>
```
*ตัวอย่าง:*
```bash
git clone https://github.com/username/my-ecommerce.git
```
*(เมื่อโคลนโปรเจกต์เสร็จสิ้นแล้ว อย่าลืมพิมพ์ `cd my-ecommerce` ใน Terminal เพื่อเปลี่ยนโฟลเดอร์ทำงานหลัก)*

---

### 2. การดึงข้อมูลอัปเดตล่าสุด (Pull)
ใช้สำหรับดึงโค้ดล่าสุดที่คนในทีมอัปเดตขึ้นสู่เซิร์ฟเวอร์กลางลงมาปรับปรุงโค้ดในเครื่องของเรา (แนะนำให้รันคำสั่งนี้ทุกครั้งก่อนเริ่มพิมพ์โค้ดใหม่ในแต่ละวันเพื่อป้องกันข้อขัดแย้ง):
```bash
git pull origin <ชื่อ_Branch>
```
*ตัวอย่าง (ดึงข้อมูลล่าสุดจากกิ่งหลัก):*
```bash
git pull origin main
```

---

### 3. การรวมโค้ดระหว่างสองกิ่ง (Merge)
เมื่อคุณพัฒนาฟีเจอร์แยกไว้ในกิ่งย่อย (เช่น กิ่ง `feature-ui`) เสร็จแล้ว และต้องการรวมข้อมูลเข้ามายังกิ่งทำงานหลัก (`main`):

1. ย้ายตัวเองกลับมาทำงานบนกิ่งหลักที่เราต้องการนำโค้ดไปรวมก่อน:
   ```bash
   git checkout main
   ```
2. ดึงโค้ดเวอร์ชันล่าสุดของกิ่งหลักมาทับเพื่อความอัปเดต:
   ```bash
   git pull origin main
   ```
3. รวมโค้ดจากกิ่งย่อยที่เราต้องการนำมารวมเข้ากับกิ่งปัจจุบัน:
   ```bash
   git merge <ชื่อ_Branch_ย่อย>
   ```
   *ตัวอย่าง:*
   ```bash
   git merge feature-ui
   ```

💡 **กรณีระบบแจ้งเตือนข้อขัดแย้ง (Merge Conflict):**
หากมีการแก้ไขไฟล์เดียวกันและบรรทัดเดียวกันในทั้งสองกิ่ง Git จะแจ้งเตือนข้อขัดแย้ง ให้เราดำเนินการดังนี้:
1. เปิดไฟล์ที่มีปัญหานั้นบนโปรแกรม Code Editor (เช่น VS Code จะแสดงไฮไลต์จุดชนกันของโค้ด)
2. ตัดสินใจเลือกว่าจะใช้โค้ดของฝ่ายใด (Accept Current / Incoming / Both Changes)
3. เมื่อแก้ไขจุดขัดแย้งเสร็จแล้ว ให้กดบันทึกไฟล์ (Save) แล้วใช้คำสั่งนี้เพื่อยืนยันการรวมงาน:
   ```bash
   git add .
   git commit -m "Resolve merge conflicts"
   git push origin main
   ```
