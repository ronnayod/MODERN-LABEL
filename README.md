# MODERN LABEL - E-Commerce Streetwear Web Application

โปรเจกต์นี้เป็นเว็บไซต์ร้านค้าออนไลน์ (E-Commerce) สำหรับแบรนด์แฟชั่นสตรีทแวร์แนวพรีเมียม ชื่อว่า **MODERN LABEL** พัฒนาขึ้นโดยใช้ Vue 3 และ Vite พร้อมกับการดีไซน์สไตล์มินิมอลและรองรับการแสดงผลแบบ Responsive เต็มรูปแบบ

---

## 🛠️ เทคโนโลยีและเครื่องมือที่ติดตั้งในโปรเจกต์ (Dependencies)

โปรเจกต์นี้ขับเคลื่อนด้วยไลบรารีและเทคโนโลยีหลักดังต่อไปนี้:

### 1. **Core Framework & Build Tools**
*   **Vue 3 (`vue`: `^3.5.34`)**: เฟรมเวิร์กหลักที่ใช้พัฒนาเว็บแบบ Single Page Application (SPA) ด้วยฟีเจอร์ Composition API (`<script setup>`)
*   **Vite (`vite`: `^8.0.12`)**: เครื่องมือสำหรับ Bundle และรันเซิร์ฟเวอร์พัฒนาที่มีความเร็วสูงและสนับสนุน Hot Module Replacement (HMR)
*   **@vitejs/plugin-vue (`^6.0.6`)**: ปลั๊กอินอย่างเป็นทางการของ Vite เพื่อสนับสนุนการคอมไพล์ไฟล์ Vue Single File Components (SFCs)

### 2. **Styling & Components**
*   **Bootstrap 5 (`bootstrap`: `^5.3.8`)**: CSS Framework สำหรับจัดการโครงสร้าง Grid, เลย์เอาต์ และดีไซน์เริ่มต้นที่รองรับ Responsive
*   **Bootstrap Vue Next (`bootstrap-vue-next`: `^0.45.6`)**: ชุดคอมโพเนนต์สำหรับ Vue 3 ที่แปลงคอมโพเนนต์เดิมของ Bootstrap 5 ให้ใช้งานง่ายขึ้นในรูปแบบ Vue Components (เช่น `BOffcanvas`)
*   **Vanilla CSS (`src/style.css`)**: มีการสร้างระบบ Design Token, ธีมสี (Light & Dark), อนิเมชันการเลื่อน, เงา และปุ่มกดแนวพรีเมียมเฉพาะตัวแบบ Custom

### 3. **Routing**
*   **Vue Router 4 (`vue-router`: `^4.6.4`)**: ระบบเราเตอร์เพื่อนำทางไปยังหน้าต่าง ๆ ของเว็บไซต์อย่างราบรื่น (เช่น หน้าหลัก หน้าหมวดหมู่สินค้า หน้าดูรายละเอียดสินค้า)

## 🚀 วิธีการรันโปรเจกต์

หากต้องการเริ่มต้นใช้งานและพัฒนาโปรเจกต์ต่อ ให้ปฏิบัติตามคำสั่งต่อไปนี้:

## 🐙 วิธีใช้งาน Git และอัปโหลดโค้ดขึ้น GitHub

โปรเจกต์นี้ใช้ **Git** ในการควบคุมเวอร์ชันของซอร์สโค้ด (Version Control) และเก็บโค้ดไว้บนคลังออนไลน์ของ **GitHub** ที่ลิงก์: [ronnayod/MODERN-LABEL](https://github.com/ronnayod/MODERN-LABEL.git)

ด้านล่างนี้คือสรุปขั้นตอนและคำสั่งในการจัดการโค้ดให้เข้าใจและใช้งานได้ง่าย:

---

### 1. 🚀 การเชื่อมต่อกับคลังบน GitHub ครั้งแรก (First-Time Setup)

#### 🔹 กรณีที่ 1: การเริ่มสร้าง Git คลังใหม่ในโฟลเดอร์เครื่องตัวเอง
หากคุณสร้างโปรเจกต์ขึ้นมาใหม่ในเครื่อง และต้องการนำขึ้น GitHub เป็นครั้งแรก:
```bash
# 1. เขียนชื่อหัวข้อโปรเจกต์ลงไฟล์ README.md
echo "# MODERN-LABEL" >> README.md

# 2. เริ่มต้นระบบ Git ในโฟลเดอร์โปรเจกต์ของคุณ
git init

# 3. นำไฟล์ README.md เข้าระบบคิวเตรียมรอบันทึก
git add README.md

# 4. บันทึกโค้ดเวอร์ชันแรกอย่างเป็นทางการลงในระบบ Git เครื่องตัวเอง
git commit -m "first commit"

# 5. ตั้งชื่อกิ่งการทำงานหลักของโปรเจกต์ให้เป็น 'main'
git branch -M main

# 6. เชื่อมโยงโฟลเดอร์เครื่องเราเข้ากับคลังเก็บโค้ดบน GitHub (Remote Server)
git remote add origin https://github.com/ronnayod/MODERN-LABEL.git

# 7. อัปโหลดโค้ดเวอร์ชันแรกขึ้นระบบ GitHub ออนไลน์
git push -u origin main
```

#### 🔹 กรณีที่ 2: การนำโฟลเดอร์ที่มีประวัติ Git อยู่แล้ว ส่งขึ้นคลัง GitHub
หากในเครื่องมีโฟลเดอร์ Git อยู่แล้ว แต่เพิ่งสร้าง Repository เปล่าบน GitHub และต้องการเชื่อมโยงเข้าด้วยกัน:
```bash
# 1. เชื่อมโยงคลัง Git ในเครื่องเข้ากับระบบออนไลน์บน GitHub
git remote add origin https://github.com/ronnayod/MODERN-LABEL.git

# 2. ตรวจสอบและตั้งชื่อกิ่งหลักเป็น 'main'
git branch -M main

# 3. อัปโหลดขึ้นสู่ GitHub พร้อมตั้งค่าความเชื่อมโยงในระยะยาว
git push -u origin main
```

### 2. 🔄 ขั้นตอนการทำงานปกติในแต่ละวัน (Daily Git Workflow)

เมื่อคุณเขียนโค้ดและแก้ไขปรับปรุงหน้าเว็บเสร็จสิ้นแล้ว และต้องการส่งงานที่อัปเดตขึ้นสู่ GitHub ให้ทำตาม 4 ขั้นตอนนี้เรียงตามลำดับ:

#### 🔍 ขั้นตอนที่ 1: ตรวจสอบสถานะการเปลี่ยนแปลง (git status)
*คำสั่งแนะนำเพื่อตรวจสอบความเรียบร้อยบ่อยๆ* เพื่อเช็คว่ามีไฟล์ใดที่เราแก้ไข เพิ่มใหม่ หรือเผลอลบไปบ้าง:
```bash
git status
```

#### 📥 ขั้นตอนที่ 2: จัดเก็บไฟล์เข้าคิวเตรียมบันทึก (git add)
เพื่อเลือกเอาไฟล์ที่เราแก้ไขนั้นเข้าไปอยู่ในคิวจัดเตรียม (การใส่เครื่องหมาย `.` หมายถึงหยิบไฟล์ทั้งหมดในโฟลเดอร์ที่เกิดการเปลี่ยนแปลงเข้าคิว):
```bash
git add .
```

#### ✍️ ขั้นตอนที่ 3: บันทึกข้อมูลประวัติ (git commit)
ทำการจดบันทึกบันทึกรายละเอียดสั้นๆ ว่าในเวอร์ชันนี้เราแก้ไขหรืออัปเดตอะไรไปบ้าง (ช่วยให้ทำงานร่วมกันและย้อนประวัติได้ง่าย):
```bash
git commit -m "setup initial project structure"
```

#### ⬆️ ขั้นตอนที่ 4: อัปโหลดโค้ดขึ้นระบบ GitHub ออนไลน์ (git push)
ส่งข้อมูลโค้ดที่เรา Commit เรียบร้อยแล้วในเครื่องขึ้นไปแสดงผลและอัปเดตบนคลัง GitHub ออนไลน์:
```bash
git push
```

### 3. 👥 การทำงานร่วมกันในทีม (Collaborating)

#### ⏬ การดาวน์โหลดโค้ดโปรเจกต์ลงเครื่องครั้งแรก (Clone)
ใช้ดาวน์โหลดโฟลเดอร์โปรเจกต์เวอร์ชันอัปเดตทั้งหมดจาก GitHub ลงเครื่องตัวเองในครั้งแรกที่เข้าร่วมทีม:
```bash
git clone https://github.com/ronnayod/MODERN-LABEL.git
```
*(เมื่อคำสั่งโคลนโปรเจกต์เสร็จสิ้นแล้ว อย่าลืมพิมพ์ `cd my-ecommerce` ใน Terminal เพื่อเปลี่ยนโฟลเดอร์ทำงานหลัก)*

#### 🔄 การดึงข้อมูลอัปเดตล่าสุดจากคลังออนไลน์ (Pull)
เพื่ออัปเดตโค้ดในเครื่องของเราให้เท่ากับโค้ดล่าสุดที่สมาชิกคนอื่นในทีมเพิ่งอัปโหลดขึ้นไป (แนะนำให้พิมพ์คำสั่งนี้ในหน้างานกิ่ง main ทุกเช้าก่อนเริ่มเขียนโค้ดใหม่เพื่อความราบรื่น):
```bash
git pull origin main
```

### 4. 🔀 การรวมโค้ดและการแก้ไขโค้ดชนกัน (Merge & Resolve Conflicts)

เมื่อคุณแก้ไขฟีเจอร์ต่าง ๆ แยกไว้ในกิ่งทำงานอื่น (เช่นกิ่ง `feature-ui`) เสร็จแล้ว และต้องการส่งมารวมกับกิ่งหลัก (`main`):

1.  ย้ายการทำงานหลักในเครื่องกลับมายังกิ่งหลัก:
    ```bash
    git checkout main
    ```
2.  ดึงโค้ดล่าสุดบนคลาวด์ลงมาอัปเดตให้เรียบร้อย:
    ```bash
    git pull origin main
    ```
3.  สั่งรวมโค้ดจากกิ่งฟีเจอร์ย่อยของเราเข้ากับกิ่งหลักปัจจุบัน:
    ```bash
    git merge feature-ui
    ```

💡 **กรณีระบบตรวจพบโค้ดทับซ้อนกัน (Merge Conflict):**
หากสองคนแก้ไขไฟล์และบรรทัดเดียวกัน Git จะให้เราช่วยตัดสินใจเลือกโค้ดด้วยตัวเองดังนี้:
1.  เปิดไฟล์ที่เกิดข้อขัดแย้งในโปรแกรมเขียนโค้ด (เช่น VS Code จะแสดงไฮไลต์ให้เห็นจุดที่มีปัญหาอย่างชัดเจน)
2.  คลิกเลือกว่าจะคงโค้ดส่วนใดไว้ (Accept Current Change / Accept Incoming Change / Keep Both)
3.  หลังจากจัดการรวมโค้ดเสร็จแล้ว ให้เซฟไฟล์ (Save) แล้วส่งอัปเดตขึ้นคลังปกติ:
    ```bash
    git add .
    git commit -m "Resolve merge conflicts"
    git push
    ```
